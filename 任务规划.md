# 任务规划：专辑封面扩产至每分类 2000 张

## 当前状态

| 项目 | 当前 | 目标 |
|------|------|------|
| 分类数量 | 10 个 | 10 个（不变） |
| 每分类图片数 | 100 张 | 2000 张 |
| 总计 | 1000 张 | 20,000 张 |

## Agent 集群分工

每个分类派一个 agent，共 10 个 agent 并行工作：

```
Agent 1  → lofi
Agent 2  → minimal
Agent 3  → collage
Agent 4  → psychedelic
Agent 5  → sketch
Agent 6  → gothic
Agent 7  → nature
Agent 8  → glitch
Agent 9  → ugly-design
Agent 10 → zen
```

## 工作流程（每个 agent 执行以下步骤）

### 第一步：阅读文档

- 阅读 `stepfun_api.md`，了解 API 调用方式
- 阅读 `outline.md`，理解风格定义和提示词指南

### 第二步：生成大图（1024×1024）

- 使用 `generate_covers_100.py` 作为模板
- 修改循环次数：每个分类从现有 100 张继续生成到 2000 张（需新增 1900 张/分类）
- 输出路径：`album-covers/{style}/{style}_{N}.png`（N 从 101 开始编号）
- 每个 prompt 随机组合 subjects + moods + colors，多样性最大化
- 每次 API 调用间隔 1-2 秒，避免限流
- **不要覆盖已有图片**：检查文件是否存在，跳过已有

**API 参考（来自 stepfun_api.md）：**
```
URL: https://api.stepfun.com/v1/images/generations
Model: step-image-edit-2
Parameters: steps=8, response_format=b64_json, text_mode=true
```

### 第三步：生成缩略图（256×256）

- 修改 `generate_thumbnails.py`，将输出路径改为 `thumbnails/{style}/`
- 把 `album-covers/{style}/` 下新生成的大图缩放到 256×256
- 输出到 `thumbnails/{style}/{style}_{N}.png`
- **跳过已存在的缩略图**（幂等）

### 第四步：更新 manifest.json

- 修改 `build_manifest.py`，输出路径改为 `manifest.json`
- 重新扫描 `album-covers/` 和 `thumbnails/` 目录
- 生成新的 manifest.json

## Agent 参考文件清单

每个 agent 需要读取以下文件：

| 文件 | 用途 |
|------|------|
| `stepfun_api.md` | API 调用方法 |
| `outline.md` | 风格定义与提示词 |
| `generate_covers_100.py` | 生成图片的 Python 脚本模板 |
| `generate_thumbnails.py` | 生成缩略图的 Python 脚本模板 |
| `build_manifest.py` | 重建 manifest.json 的 Python 脚本模板 |

## 注意事项

1. **不覆盖已有文件**：生成前先检查文件是否存在
2. **编号连续递增**：从当前最大编号 +1 开始
3. **限流保护**：API 调用间隔 ≥ 1 秒
4. **错误重试**：网络错误时 sleep 5 秒后重试，最多 3 次
5. **不要提交到 git**：所有 agent 完成后，由人工审查再决定是否推送
